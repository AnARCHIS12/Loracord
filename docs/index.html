Features

Low-bandwidth social mesh, zero central server

01

Guilds and channels

Create communities, add custom text channels, invite by code, and keep local history.

02

Direct messages

Private conversations by lightweight user ID, carried through the Meshtastic private app port.

03

Store and forward

History catch-up through compact sync requests between nodes that already hold messages.

04

Encryption

Meshtastic PSK for the LoRa channel, Loracord AES-GCM 256, and X25519 DM invites.

Supported hardware

Any BLE Meshtastic node can carry Loracord

Loracord currently connects from Android through the official Meshtastic Bluetooth LE client service. Use nodes with BLE enabled and the same Meshtastic channel settings.

RAK WisBlock

RAK4631, WisMesh Pocket, Pocket Mini, Tag, and TAP.

Heltec LoRa

LoRa 32 V3/V4, Wireless Stick Lite V3, and trackers.

Heltec T114

Low-power nRF52 node with BLE and modern LoRa radios.

SenseCAP

T1000-E and Wio Tracker L1 devices with Meshtastic firmware.

LILYGO T-Echo

Portable BLE Meshtastic node with screen and GNSS options.

LILYGO T-Deck

T-Deck, T-Deck Plus, T-Beam S3, and compatible BLE boards.

Not supported yet: USB-only serial nodes, Wi-Fi/TCP-only access, and devices without Bluetooth LE.

Architecture

The phone drives, the mesh transports

Loracord encodes compact binary frames, fragments short messages, then sends them to a Meshtastic node over BLE. The ESP32/radio node handles mesh transmission across LoRa.

  • Transport Android BLE Meshtastic ToRadio / FromRadio
  • Port Meshtastic PRIVATE_APP
  • Loracord payloads encrypted before ToRadio
  • DM invites with X25519 key derivation
  • Loracord fragmentation and reassembly
  • Notifications locales Android
  • Fully offline operation
